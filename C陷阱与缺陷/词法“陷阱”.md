# 词法陷阱

**符号**：（token）指的是程序的一个基本组成单元，其作用相当于句子中的单词。编译器中负责将程序分解为一个一个符号的部分，一般称为“词法分析器”。

## = 和 ==

在C语言中`=` 是复制运算符，`==` 是比较运算符。由于数学上的定式，很多时候在写if语句的时候容易出错，比如：

```c
if (x = y){
    break;
}
```

程序的本意是：判断x是否等于y，然而实际上，这条语句的含义却是，将y赋值给x，然后判断x是否为0或其他数。

另外一种情况，如果我确实需要去判断，`x = y` 之后x的值是否为0的情况，比如：

```c
if (x = y){                               // no
    foo();
}
```

这种写法，在一些编译器中就会抛出警告，因此，在应用中，如果确实存在这样的需求，应当显式的进行比较：

```c
if ((x = y) != 0){                        // yes
    foo();
}
```

## | 、& 和 || 、&&

`|` : 按位或

`&` ：按位与

`||` ：逻辑或

`&&` ：逻辑与

实际应用中逻辑运算符较为常用，不要写成位运算符即可。

## 词法分析之“贪心法”

编译器如何读入字符之后判断该字符是独立存在还是和之前或者之后的字符能够组成关键字呢？

实际上编译器采用的就是一种贪心法，也就是尽可能多的读入，比如：`for (int i = 0;i < n;i ++){}` 编译器会依次读入f、o、r...每读入一个字符，编译器就会和前面的字符结合进行判断，是否可以组成关键字，当for已经读入完成，编译器发现for已经是关键字了，这时候会再读入下一个字符进行判断，如果下一个字符与for没有关系，那么下一个字符，就是下一次贪心的起点，for关键字就成功读入了。

### 自增自减

经典疑问：`a---b`是什么意思？

正确答案是：`a-- - b` 

错误答案是：`a - --b`

why？C语言中，运算符是有优先级的，而自增自减的优先级是高于加减运算符的优先级的。所以在编译器按照贪心法读取语句符号的时候，读入第一个`-`号，（此时建好已经有意义了）编译器贪心，并未满足，会继续读入下一个`-`号，然后发现，哎，我又变成自增了（这里之所以会变，就是因为优先级的问题），我还是有意义的，那么继续读，知道读入下一个`-`号，C语言中`---`这东西可是没有意义的，因此，前面的两位组成自减，后面的一位作为减号。

### 除以指针变量

经典疑问：`y = x/*p` 运行结果是怎样的？

正确答案是：x 赋值给 y。

错误答案是：x 除以指针变量指向的值

why？同样是贪心法，编译器读取了`/`之后并没有直接认为他是除号，而是继续往下读取了一位，这就变成了`/*`，哎，这就变成注释了呀！

正确的写法应该是：`y = x / (*p);`

### += 不要写成 =+

很多的算法中会涉及到这类组合运算符，在较老的编译器中，`+=`和`=+`都是可以编译通过的，但是较新的编译器就会出现歧义。而且如果没有空格在符号两侧，则会更糟糕！

`x=-1` 对于这段代码，相信大多数人的理解都是：`x =- 1` 。然而实际上，程序员本身想要表达的意思是：

`x = -1`，在双向运算符左右敲空格是个好习惯，另外+=不能再写成=+了

### 整型常量

在C语言中，以数字0开头的数字被认为是八进制，这会很容易在程序中出现错误：

```c
struct {
    int part_number;
    char *description;
}parttab[] = {
    046,	"left-handed widget"	,
    047,	"right-handed widget"	,
    125,	"frammis"	
};
```

上面的程序段中，本想着将十进制数46写成046以对齐代码，没想到会弄巧成拙，直接变成八进制数了。

### 字符和字符串

C语言中的单引号和双引号含义是不一样的。用单引号引起的一个字符实际上代表一个整数，整数值对应于该字符在编译器采用的字符集中的序列值。双引号引起的字符串，代表的却是一个指向无名数组起始字符的指针。
